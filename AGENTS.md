# diegocasmo.github.io

Personal blog by Diego Castillo. Built with Astro 7, TypeScript, vanilla CSS. Deployed to GitHub Pages.

> `CLAUDE.md` is a symlink to this file ‚Äî edit here.

## Philosophy & Principles

- **SEO is non-negotiable** ‚Äî never degrade existing meta, Open Graph, structured data, or social sharing tags
- **Accessibility (WCAG 2.1 AA)** ‚Äî semantic HTML, ARIA attributes, keyboard navigation, visible focus management
- **Mobile-first responsive design** ‚Äî single breakpoint at `max-width: 684px` (plus a small `@media print` stylesheet)
- **Readability** ‚Äî monospace typography (Fira Code), high contrast, generous `line-height: 1.54em`
- **Social sharing quality** ‚Äî OG tags and Twitter cards on every page; `og:image` is auto-generated for all posts via `/og/[...slug].png.ts`, and every static/listing page falls back to a site card generated by `/og/default.png.ts` (default set in `BaseLayout.astro`)
- **Agent experience (AEX)** ‚Äî the site is machine-readable for LLMs/agents: `/llms.txt` (post index), `/llms-full.txt` (all posts concatenated), and a `.md` Markdown mirror of every post at `/{slug}.md`, all generated from the content collection at build time. Keep them in sync with content by never hardcoding post lists
- **Minimal dependencies** ‚Äî vanilla CSS, no frameworks, no unnecessary JS; the site is intentionally static
- **Content direction** ‚Äî evolving from web dev toward systems programming, CS fundamentals, and low-level engineering

## Content Schema

Post frontmatter (`src/content.config.ts`):

```yaml
title: string          # Required
description: string    # Required
pubDate: date          # Required
updatedDate: date      # Optional
image: string          # Optional
externalLink: string   # Optional
tags: string[]         # Default: []
draft: boolean         # Default: false ‚Äî drafts are filtered from all listings
```

Slugs are date-prefix-free. Legacy redirects (date-prefixed and `/posts/` paths) are mapped in `astro.config.mjs`.

## SEO & Structured Data ‚Äî Must Preserve

- **Canonical URLs** ‚Äî generated in `BaseLayout.astro`
- **Open Graph** ‚Äî `og:title`, `og:description`, `og:type`, `og:url`, `og:image`, `og:site_name`, `og:locale`, plus `article:*` on posts
- **Twitter Cards** ‚Äî `summary_large_image`, `twitter:site`, `twitter:creator` (both must include the `@`)
- **Schema.org (JSON-LD)** ‚Äî this is the canonical structured-data format for the site (Google recommends JSON-LD; do not reintroduce microdata). Injected via the `jsonLd` prop on `BaseLayout.astro`:
  - `BlogPosting` on every post (`PostLayout.astro`) ‚Äî headline, description, image, datePublished, dateModified (only when `updatedDate` is set), author + publisher (Person), keywords, mainEntityOfPage
  - `WebSite` + `Person` on the homepage (`index.astro`) ‚Äî `Person` carries `sameAs`, `jobTitle`, `worksFor`, `alumniOf`, `knowsAbout`; shares a stable `@id` (`#person`) with the about page
  - `ProfilePage` (mainEntity ‚Üí `Person`) on `/about` (`about.astro`)
  - When Diego's role/employer changes, update the `Person` facts in `index.astro` **and** `about.astro`
- **Sitemap** ‚Äî `@astrojs/sitemap` at `/sitemap-index.xml` (non-page endpoints like `.md`/`.txt`/og `.png` are excluded automatically)
- **RSS** ‚Äî `@astrojs/rss` at `/rss.xml`
- **robots.txt** ‚Äî in `public/`; blanket-allow (correct for AI-answer visibility), `Sitemap:` points at `/sitemap-index.xml`
- **OG images** ‚Äî auto-generated at build time via `src/pages/og/[...slug].png.ts` (per-post) and `src/pages/og/default.png.ts` (site fallback for static pages), using satori + @resvg/resvg-js; override per-post with the `image` frontmatter field
- **IndexNow** ‚Äî on deploy, `.github/workflows/deploy.yml` pings IndexNow (Bing/Copilot, Yandex, et al.) with the live sitemap URLs; the key lives in `public/<key>.txt` (public by design)

## Animations

Interactive algorithm visualizations used in `.mdx` posts. Components live in `src/components/animations/`.

> `.mdx` is enabled via `@astrojs/mdx`; currently only `merge-sort.mdx` uses it (the other 29 posts are plain `.md`).

**Architecture:**
- `AlgoViz.astro` ‚Äî generic player shell (play/pause/replay controls, progress bar, accessibility announcements, `prefers-reduced-motion` support). Wraps algorithm content via `<slot />`
- `ArrayGroup.astro` ‚Äî reusable SVG primitive for rendering labeled number arrays (bounding box + cells). Use `showCells={false}` when animating individual cells independently
- Algorithm components (e.g., `MergeSort.astro`) ‚Äî pure SVG + CSS keyframe animations, rendered inside `AlgoViz`

**Authoring a new animation:**
1. Create `src/components/animations/<Algorithm>.astro` with a `step` prop for multi-scene animations
2. Use `ArrayGroup` for array visualizations; write raw SVG for other structures
3. Define all animations as CSS keyframes with explicit `ms` durations and delays ‚Äî no JS animation libraries
4. Use the post's `.mdx` format and import both `AlgoViz` and the algorithm component
5. Wrap each scene: `<AlgoViz title="..." id="..." duration={totalMs}><Algorithm step={n} /></AlgoViz>`
6. Set `duration` to match the longest animation delay + duration in the CSS for that step

## Git & Deploy

- **Deploy**: GitHub Actions on push to `main` (`.github/workflows/deploy.yml`) ‚Äî Node 22 ‚Üí `npm ci` ‚Üí `npm run build` ‚Üí `actions/upload-pages-artifact@v3` ‚Üí `actions/deploy-pages@v4`
- **CI**: `.github/workflows/ci.yml` runs on push and PR to `main` ‚Äî `npm run build` then `npm test` (Playwright/Chromium). PRs fail if either breaks.
- **Build pipeline**: `astro check` runs before `astro build` (via the `build` script)
- **Dependabot** (`.github/dependabot.yml`): daily direct-dep npm bumps, 2-day cooldown, max 3 open PRs, commit prefix `chore(package.json):`

## Boundaries

- **Do not** add UI frameworks (React, Vue, Svelte) ‚Äî the blog is intentionally framework-free
- **Do not** add CSS frameworks or preprocessors
- **Do not** modify redirect mappings in `astro.config.mjs` without understanding the legacy URL structure
- **Do not** remove or weaken any SEO meta tags, structured data, or social sharing tags; JSON-LD is the canonical structured-data format (do not revert to microdata)
- **Do not** hardcode post lists in `llms.txt.ts`, `llms-full.txt.ts`, or `[...slug].md.ts` ‚Äî they must derive from `getCollection('posts')` so they stay in sync
- **Do not** use JS animation libraries (GSAP, Framer Motion, etc.) ‚Äî animations must be pure CSS keyframes
- **Do not** bypass `AlgoViz.astro` for interactive animations ‚Äî it provides play/pause, progress tracking, and accessibility (`prefers-reduced-motion`, ARIA live regions)
- **Do not** add JS-based image generation at runtime ‚Äî OG images are generated statically at build time
